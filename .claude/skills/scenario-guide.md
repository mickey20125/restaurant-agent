## 一、執行步驟

收到使用者查詢後，依序完成以下步驟，**然後才呼叫 pipeline**：

1. **識別情境**：從 prompt 判斷屬於哪個用餐場合
2. **展開調性**：同一情境主動拆出 2–3 種調性，一次滿足不同需求
3. **解碼模糊指令**：把「有質感」「安全牌」等主觀詞彙翻譯成可操作的篩選條件
4. **解析飲食歷史**（條件觸發）：**只有當使用者在 prompt 中提及「最近吃過什麼」或「不吃什麼」時才執行**，將吃過的內容展開為排除條件
5. **判斷是否需要追問**：資訊充足則直接推薦；缺少關鍵資訊則先追問（只問一個最關鍵的問題）

## 二、調性維度系統

同一情境下，不同使用者期待可能截然不同。AI 應主動在一次回答中展開 2–3 種調性選項；若使用者已明確表達調性（例如「浪漫一點」），則直接依該調性推薦。

### 通用調性維度表

每次推薦前，對照下列四個維度，挑出 1–2 個在該情境容易產生分歧的維度來展開：

| 維度 | 選項 A | 選項 B | 選項 C |
|------|--------|--------|--------|
| 氛圍 | 安靜沉穩 | 輕鬆活潑 | 獨特個性 |
| 空間感 | 私密包廂型 | 開放大廳型 | 戶外或半戶外 |
| 體驗形式 | 坐下來好好吃 | 邊走邊吃／探索型 | 有互動儀式感（吧台、主廚桌） |
| 價位感 | 高（儀式感） | 中（日常好店） | 低（意外驚喜） |

## 三、情境識別與 constraints

### 社交場合類

#### 約會
觸發詞：約會、帶另一半、紀念日、求婚、浪漫

核心 constraints：桌距寬鬆、燈光偏暖、背景音量適中、避免需豪邁動手的食材（螃蟹、龍蝦）、服務細心但不過度打擾

排除：大排檔、嘈雜開放式廚房、明亮螢光燈環境

調性展開：
- **安靜浪漫型**：Fine dining 或低調日料，燈光昏暗、桌距寬鬆
- **輕鬆探索型**：市場小吃、居酒屋或主題餐廳，重點是輕鬆互動
- **儀式感型**：吧台席、主廚桌或無菜單料理，適合想創造共同回憶者

#### 帶長輩
觸發詞：長輩、爸媽、阿公阿嬤、年長、父母

核心 constraints：交通便利、有停車、環境安靜、座椅舒適、菜單有清淡選項

排除：久候排隊、地下室或多層樓梯、過度時髦形式

調性展開：養生清淡型／傳統台味型／熱鬧家庭型

#### 見家長（第一次）
觸發詞：見家長、第一次見父母

核心 constraints：口碑穩定需先訂位、優先包廂或半私密空間、菜系大眾接受（台菜、粵菜、日式）、價位合宜

排除：網紅排隊店、過度創意料理、自助餐、熱炒

調性展開：穩重傳統型／精緻日式型／輕鬆家常型

#### 第一次見網友
觸發詞：見網友、第一次見面

核心 constraints：公開且人多、時間控制 1–1.5 小時、AA 結帳容易、周邊有其他活動選項

排除：包廂、無菜單料理（時間不可控）

調性展開：低壓安全型／話題製造型／時間可控型

#### 部門／團體聚餐
觸發詞：部門聚餐、團隊、同事、尾牙、慶功

核心 constraints：有大桌或包廂、菜色多元（顧及素食/過敏）、不強迫互動形式

排除：無菜單只有一種選項的餐廳

調性展開：熱鬧桌菜型／各點各的定食型／有包廂儀式感型

## 四、模糊指令解碼表

| 使用者說的 | 實際需求 | 可操作的篩選條件（加入 --logic） |
|-----------|---------|-------------------------------|
| 有感覺的店 | 氛圍對 | 燈光暖色/昏暗、音量不吵、裝潢有主題、桌距不擁擠 |
| 有質感的店 | 細節用心 | 餐具精緻、擺盤整齊、服務訓練良好 |
| 厲害的店 | 有招牌必吃菜 | 確認有值得被討論的招牌菜 |
| 吃好吃的 | 犒賞自己，不計較性價比 | 食材等級高、主廚資歷、用料紮實 |
| 隱藏店 | 口耳相傳 | 位置不顯眼、回頭客多、非網路行銷導向 |
| 安全牌 | 不會踩雷 | 口碑穩定、菜系大眾接受、評論數足夠且近期無明顯負評 |

## 五、飲食歷史解析（條件觸發）

**觸發條件**：僅在使用者明確提及下列情況時啟動：
- 情況 A：提到「最近吃過什麼」→ 展開為排除條件，避免重複推薦
- 情況 B：明確說「不吃／不要／過敏」某類食材 → 展開為硬性排除

**不要主動詢問**「你最近吃過什麼」或「有什麼不吃的」；沒有提及就直接進推薦流程。

展開規則：
- 「吃過」→ `exclude_cuisine`（避免重複，時間越近權重越高）
- 「不吃／過敏」→ 硬性排除，並在輸出中提醒使用者向餐廳確認食材
- 在輸出文字中說明：「你昨天剛吃過義式，我幫你避開這類選項。」

## 六、輸出格式（傳給 pipeline 的參數）

scenario-guide 完成後，輸出以下參數供 `uv run python -m tools.cli agent --markdown` 使用：

```
--query  "<情境描述 + 解碼後的具體需求>"
--logic  "<情境 constraints + 排除條件 + 調性描述>"
--top-k  <依調性數量決定，通常 1-2 間每種調性>
```

範例（約會，安靜浪漫型）：
```bash
uv run python -m tools.cli agent \
  --query "約會，步行 20 分鐘內，日式或法式" \
  --logic "桌距寬鬆、燈光偏暖、背景音量適中、避免螃蟹龍蝦等需大動作的食材、服務細心、安靜浪漫型" \
  --user-lat 25.047094 --user-lng 121.542698 \
  --top-k 2 --markdown
```

## 七、AI 行為準則

1. 不只推「評分最高」的店，應依情境挑選最適合的店
2. 預設展開 2–3 種調性，讓使用者有選項
3. 調性標籤要描述性（「安靜沉穩型」），不要評價性（「高級型」）
4. 情境衝突時說出來，請使用者選擇
5. 模糊指令先解碼再推薦，或解碼後向使用者確認
6. 「不要網美店」不等於不能好拍，優先食物品質
7. 「安全牌」與「不要一次店」合併理解
8. 帶長輩重體貼（無障礙、清淡），見家長重表演（口碑、訂位、節奏）
9. 追問只問一個問題
10. 飲食歷史解析只在使用者有提及時才啟動
11. 「不吃」的排除權重高於「吃過」
